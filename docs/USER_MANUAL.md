# 동양구조 업무관리 — 사용 매뉴얼

> 사내 임직원용. 개발자가 아닌 일반 사용자 기준으로 작성.

## 1. 접속

브라우저(권장: Chrome / Edge 최신)에서 **https://task.dyce.kr** 접속.

설치할 앱은 없습니다. 사내 어디서든 인터넷만 되면 같은 화면을 사용할 수 있습니다.

## 2. 첫 사용 (로그인)

### 2.1 NAVER WORKS 로그인 (권장)
로그인 화면에서 **「NAVER WORKS로 로그인」** 클릭 → 회사 NAVER WORKS 계정으로 인증 → 자동 가입.

### 2.2 가입 신청 (비-NAVER WORKS 계정)
별도 가입이 필요한 경우 로그인 화면에서 **「가입 신청」** 클릭 → 이름/이메일 입력 → **관리자 승인 후** 로그인 가능합니다.

> 본인 이름은 **노션 담당자 옵션과 정확히 일치**해야 본인 프로젝트가 「내 업무」에 표시됩니다. 다르면 우상단 프로필에서 수정.

## 3. 주요 화면

### 3.1 대시보드 (관리자/팀장)
- **월별 추이 차트**: 수주액·수금액·지출액 막대 + 12개월 선형회귀 추세선
- **진행단계 칸반**: 진행중·대기·보류·완료·타절·종결·이관 7개 컬럼
  - 카드 드래그로 단계 변경 (관리자만)
  - 완료/타절/종결로 옮기면 완료일·금액 입력 모달이 자동으로 뜸
- **팀별/직원별 부하 히트맵**, 업무유형 매출 트리맵, 현금흐름 예측, 지출 구분 추이
- 일반 직원은 대시보드에 진입할 수 없으며 「내 업무」로 자동 이동

### 3.2 프로젝트 목록
- 검색 / 단계 / 팀 / 완료 여부 필터
- 카드 클릭 → 프로젝트 상세

### 3.3 프로젝트 상세
- 제목 옆 **「편집」** / **「🔖 날인요청」** 버튼
- 라이프사이클 타임라인 (수주 → 계약기간 → 완료, 마일스톤)
- 진행률 게이지 (프로젝트 진척 + 수금률)
- 현금흐름 (수금/지출 누적 + 용역비 목표선)
- 지출 구분 도넛 (Top 5)
- **업무 TASK 칸반** (시작 전 / 진행 중 / 완료 / 보류) — 카드 드래그로 상태 변경
- 우상단 **「노션에서 열기 ↗」** 로 원본 페이지 이동
- WORKS Drive 연결 시 **「폴더 보기」** / **「WORKS Drive」** / **「PC 경로 복사」**

### 3.4 내 업무 (`/me`)
- 본인이 담당자로 등록된 프로젝트만 표시
- 마감 임박 TASK D-day 정렬
- **「+ 새 프로젝트」** / **「프로젝트 가져오기」** (다른 프로젝트의 담당자로 본인 추가)

### 3.5 날인요청
- 프로젝트 상세 화면의 **「🔖 날인요청」** 버튼에서만 등록 가능
- 검토구분: 구조계산서 / 구조안전확인서 / 구조검토서 / 구조도면 / 보고서 / 기타
- 구조검토서는 **문서번호 자동 발급** (예: `26-의견-057`)
- 등록 시 **검토자료 폴더** 자동 안내 — `「📁 폴더 생성」` 클릭 시 NAVER WORKS Drive에 `0.검토자료/오늘날짜/` 폴더 생성, 이후 **「📁 폴더 열기」** 로 임베디드 탐색기 진입(파일 업로드/삭제 가능)
- 등록 시 폴더가 없거나 비어있으면 확인 알림이 나옴

## 4. 자주 쓰는 작업

### 4.1 새 프로젝트 만들기
1. 「내 업무」 → **「+ 새 프로젝트」**
2. 프로젝트명 / Sub CODE / 발주처 / 수주일 / 용역비·VAT 입력
3. 발주처가 자동완성 목록에 없으면 **「발주처 DB에 추가」** 클릭 → 노션 발주처 DB에 신규 등록
4. 담당자: 본인이 default. 추가 인원 입력 가능 (콤마/엔터)
5. 업무내용: 노션 옵션에서 다중 선택 (자유 입력 가능)
6. **「생성」**

### 4.2 업무 TASK 등록
1. 프로젝트 상세 → 칸반의 **「+ 추가」** 버튼
2. 제목 / 시작일 / 마감일 / 담당자 / 우선순위 / 활동 입력
3. **「등록」**

### 4.3 날인요청 등록 (구조검토서 예시)
1. 프로젝트 상세 → **「🔖 날인요청」**
2. 검토구분: **구조검토서** 선택 → 문서번호 자동 발급 표시
3. 제출 예정일 / 내용요약 입력
4. **「📁 폴더 생성」** 클릭 → 잠시 후 **「폴더 열기」** 로 변경
5. **「폴더 열기」** → 임베디드 탐색기에서 파일 드래그 업로드
6. **「등록」** → 팀장(또는 관리자)에게 NAVER WORKS Bot 알림 발송

### 4.4 프로젝트 단계 변경 (완료 처리)
1. 프로젝트 상세 헤더의 **「단계변경」** 클릭
2. **완료** 선택 → 완료일(default 오늘) 확인 → **「저장」**
   - 타절: 타절금액 + VAT 입력
   - 종결: 용역비/VAT가 자동 ₩0

## 5. NAVER WORKS Bot 알림

날인요청의 단계별 알림이 NAVER WORKS 메신저로 자동 전송됩니다. 자세한 트리거는 [`bot_trigger.md`](./bot_trigger.md) 참고.

| 시점 | 받는 사람 |
|---|---|
| 날인요청 등록 (일반 직원) | 같은 팀의 팀장 |
| 날인요청 등록 (팀장/관리자) | 관리자 전원 |
| 1차 승인 | 관리자 전원 |
| 최종 승인 | 요청자 |
| 반려 | 요청자 (사유 포함) |

## 6. 자주 묻는 질문

### Q. 「내 업무」가 비어있어요
본인 이름이 노션 담당자 옵션과 일치하지 않을 가능성이 큽니다. 우상단 프로필에서 본인 이름을 노션과 정확히 같은 형태로 수정.

### Q. 발주처를 입력했는데 노션에 안 들어갔어요
입력한 발주처 이름이 발주처 DB에 등록되어 있지 않으면 임시 텍스트 컬럼에 들어갑니다. 입력란 아래 **「발주처 DB에 추가」** 버튼을 눌러야 정식 relation으로 저장됩니다.

### Q. 날인요청 등록 후 알림을 못 받았어요
받는 사람의 NAVER WORKS 계정이 시스템에 연결되지 않은 경우 발송이 skip됩니다. 관리자에게 본인 NAVER WORKS ID 등록 요청.

### Q. 프로젝트가 노션에는 있는데 화면에서는 안 보여요
시스템은 노션과 5분 주기로 동기화합니다. 잠시 후 새로고침해보시고, 그래도 안 보이면 우상단 메뉴의 **「동기화 강제 실행」**(관리자) 또는 관리자에게 문의.

### Q. 화면이 멈추거나 502 에러가 나와요
- 강력 새로고침: `Ctrl + Shift + R`
- 그래도 계속되면 잠시 후(1~2분) 재시도. 그래도 안 풀리면 관리자에게 문의.

## 7. 권한 안내

| 역할 | 가능한 작업 |
|---|---|
| 일반 직원 (member) | 본인 담당 프로젝트/업무 관리, 날인요청 등록 |
| 팀장 (team_lead) | + 같은 팀 직원 프로젝트 보기, 날인 1차 승인 |
| 관리자 (admin) | + 대시보드 접근, 칸반 단계 변경, 사용자 승인, 날인 최종 승인 |
