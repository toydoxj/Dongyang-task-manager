# 동양구조 업무관리 — 사용 매뉴얼

> 사내 임직원용. 개발자가 아닌 일반 사용자 기준. 앱 안에서도 좌측 사이드바의
> **사용 매뉴얼** 메뉴(`/help`)에서 같은 내용을 볼 수 있습니다.
>
> 본 문서가 **source of truth**입니다. `/help` 페이지(`frontend/app/help/page.tsx`)는 이 문서를 손으로 동기화합니다.

## 1. 접속

브라우저(권장: Chrome / Edge 최신)에서 **https://task.dyce.kr** 접속.

설치할 앱은 없습니다. 사내 어디서든 인터넷만 되면 같은 화면을 사용할 수 있습니다.

## 2. 첫 사용 (로그인)

### 2.1 NAVER WORKS 로그인 (권장)
로그인 화면에서 **「NAVER WORKS로 로그인」** 클릭 → 회사 NAVER WORKS 계정으로 인증 → 자동 가입.

### 2.2 가입 신청 (비-NAVER WORKS 계정)
별도 가입이 필요한 경우 로그인 화면에서 **「가입 신청」** 클릭 → 이름/이메일 입력 → **관리자 승인 후** 로그인 가능.

> 본인 이름은 **노션 담당자 옵션과 정확히 일치**해야 「내 업무」에 본인 프로젝트가 표시됩니다. 다르면 우상단 프로필에서 수정.

## 3. 주요 화면

### 3.1 대시보드 (관리자 / 팀장 / 관리팀)
- **월별 추이 차트**: 수주액·수금액·지출액 막대 + 12개월 선형회귀 추세선
- **진행단계 칸반**: 진행중·대기·보류·완료·타절·종결·이관 7개 컬럼
  - 카드 드래그로 단계 변경 (**관리자만**)
  - 완료/타절/종결로 옮기면 완료일·금액 입력 모달이 자동으로 뜸
  - 우측 끝 **[+ 새 프로젝트]** 컬럼 — 즉석 등록(담당자 비어있는 상태로 시작, 진행단계 = `대기`)
- **팀별/직원별 부하 히트맵**, 업무유형 매출 트리맵, 현금흐름 예측, 지출 구분 추이
- 일반 직원은 대시보드 진입 시 「내 업무」로 자동 이동

### 3.2 프로젝트 목록
검색 / 단계 / 팀 / 완료 여부 필터. 카드 클릭 → 프로젝트 상세.

### 3.3 프로젝트 상세
구성 (위→아래):
1. **헤더** — 프로젝트명 옆 **「편집」** / **「🔖 날인요청」** 버튼. 우측에 단계변경/진행단계.
2. **라이프사이클 타임라인** — 수주 → 계약기간 → 완료 + 마일스톤
3. **담당 이력 swim lane** — 담당자별 지정~해제 기간을 색 막대로. 진행 중은 끝에 →
4. **날인 현황 리스트** — 이 프로젝트의 모든 날인요청. 항목 클릭 시 상세 + 재날인요청 가능
5. **업무 TASK 칸반** — 시작 전 / 진행 중 / 완료 / 보류. 카드 드래그로 상태 변경
6. **현금흐름 / 지출 구분 도넛 등**
- WORKS Drive 연결 시 **「폴더 보기」** / **「WORKS Drive」** / **「PC 경로 복사」**

### 3.4 내 업무 (`/me`)
- 본인 담당 프로젝트만 표시. **「해야할 일」 / 「담당 프로젝트」 / 「내 영업」** 세 섹션 모두 펼치기/접기 가능
- **담당 프로젝트**는 진행 중 / 대기로 분리. 각 row 클릭 시 그 프로젝트의 TASK 칸반이 펼쳐짐
- **내 영업**도 동일 row 스타일 — 수주영업/기술지원으로 묶임. row 펼침 시 그 영업의 TASK 칸반 (영업 견적서 작성 task 자동 생성)
- 「해야할 일」의 일정 카드(외근/출장/휴가) 중 **휴가 카드 우상단**에 **「+ 새 휴가」** 버튼 — 휴가(연차) 분류로 prefilled된 신규 TASK 모달
- **「+ 새 프로젝트」** / **「프로젝트 가져오기」** (다른 프로젝트의 담당자로 본인 추가)
- **완료된 TASK는 저저번 주 월요일 이후 완료된 것까지만 표시** (오래된 완료는 자동 정리)
- 우상단 **「주간업무일지 보기」** 버튼 — 주간업무일지 페이지로 진입

### 3.5 날인요청 (목록 페이지)
- 팀장/관리자 전용. 일반 직원은 프로젝트 상세에서 본인 항목 진행상황만 확인.
- 정렬:
  - **팀장**: `1차검토 중` → `2차검토 중` → `반려` → `승인`
  - **관리자**: `2차검토 중` → `1차검토 중` → `반려` → `승인`
  - 같은 상태 안에서: 검토중·반려는 제출예정일 가까운 것 먼저, 승인은 최신 순
- 팀장은 본인 팀 직원의 요청만 보임
- **팀장은 1차 단계에서만 처리(승인·반려) 가능**, 2차는 관리자

### 3.6 주간업무일지 (`/weekly-report`)
- 진입: **대시보드 우상단** 또는 **「내 업무」 우상단**의 emerald **「주간업무일지 보기」** 버튼 (사이드바엔 없음)
- 상단 3개 날짜 입력 — 지난주 시작일 / 이번주 시작일(월요일 자동 정규화) / 이번주 종료일. 마지막 발행분 기준으로 지난주 시작일이 자동 셋팅됨
- 섹션: **인원현황** · **공지/교육/건의** · **완료 프로젝트**(상태/CODE/프로젝트명/발주처/담당팀/소요기간) · **날인대장**(저번주 승인분, 승인일순) · **영업**(수주확률 포함) · **신규 프로젝트** · **개인 주간 일정**(5팀 grid + 본부) · **팀별 업무 현황**(직원 셀병합 + 그룹 boundary) · **대기/보류 프로젝트**(자체 2-col)
- **「PDF 확인」** (admin) — 현재 입력 기간으로 PDF 빌드 후 미리보기
- **「PDF 다운로드」** (비admin) — 가장 최근 **발행된** 일지만 다운로드. 발행 이력 없으면 404
- **「발행」** (admin) — 확인 dialog 후 WORKS Drive `[주간업무일지]/YYYYMMDD_주간업무일지.pdf` 업로드 + **전 직원에게 Bot 알림 발송** ("MMDD~MMDD 주간업무일지가 업로드 되었습니다.") + 발행 로그 기록 (다음 일지 default 셋팅 기준)
- 프로젝트명 / 영업명 등의 link는 **관리자만** 활성화 — 일반 사용자에게는 plain text

## 4. 날인요청 자세히

### 4.1 등록 (프로젝트 상세 → 🔖 날인요청)
- 검토구분: 구조계산서 / 구조안전확인서 / 구조검토서 / 구조도면 / 보고서 / 기타
- 구조검토서는 **문서번호 자동 발급** (예: `26-의견-057`)
- **검토자료 폴더** 영역
  - 미생성 시: **[📁 폴더 생성]** 버튼 → `[코드]프로젝트명/0.검토자료/오늘날짜/` 자동 생성
  - 생성 후: **[📁 폴더 열기]** (임베디드 탐색기) 활성. 파일 업로드/삭제 가능
- 등록 시 폴더가 없거나 비어있으면 확인 경고
- 첨부파일 input은 폐지 — Drive에 직접 업로드

### 4.2 검토 흐름
1. 등록 → 자동으로 **요청자 TASK + 1차 검토자 TASK** 생성, NAVER WORKS Bot으로 1차 검토자에게 알림
2. **1차 승인** (팀장/관리자) → 2차검토 중으로 전환, 2차 검토자 TASK 생성, 관리자 전원에게 알림
3. **2차 승인** (관리자) → 승인 완료. 모든 TASK 완료 처리(완료일=승인일), 요청자에게 알림
4. **반려** → 현재 단계 검토자 TASK 완료. 요청자에게 사유와 함께 알림
5. **재요청** (반려 후) — 반려된 항목 클릭 → 입력 수정 → 다시 1차검토 중으로

### 4.3 재날인요청
- 프로젝트 상세 → 날인 현황에서 항목 클릭 → 상세 모달 → **[🔁 재날인요청]**
- 이전 입력 내용이 모두 prefill (구조검토서는 같은 문서번호 유지)
- **DB row를 새로 만들지 않고 같은 페이지에 덮어쓰기** + 자동 TASK 새 사이클 시작
- 라이프사이클 / 담당 이력 / 칸반이 즉시 갱신

### 4.4 날인취소
- [날인취소] 버튼 → confirm 후 실행
- 노션 페이지 archive 또는 (구조검토서 중간 번호인 경우) `[날인취소]` prefix
- 연결된 모든 TASK(요청자/1차/2차)는 **완료 상태로 자동 마감** (history 보존)

## 5. 자주 쓰는 작업

### 5.1 새 프로젝트 만들기
1. 진입점 두 가지
   - 「내 업무」 → **「+ 새 프로젝트」** (담당자 = 본인 default)
   - **대시보드 진행단계 칸반 우측 끝의 [+ 새 프로젝트] 컬럼** (담당자 비어있는 상태로 시작 — 관리자/팀장 즉석 등록용)
2. 프로젝트명 / Sub CODE / 발주처 / 수주일 / 용역비·VAT 입력
3. 발주처가 자동완성 목록에 없으면 **「발주처 DB에 추가」** 클릭 → 노션 발주처 DB에 신규 등록
4. 담당자: 콤마/엔터로 추가. **담당팀은 입력 불필요** — 노션이 자동 집계
5. 업무내용: 노션 옵션에서 **다중 선택** (자유 입력 가능)
6. **「생성」** — 신규 프로젝트의 진행단계는 `대기`로 시작 (TASK 활동이 잡히면 `진행중`으로 자동 전환)

### 5.2 업무 TASK 등록
1. 프로젝트 상세 → 칸반의 **「+ 추가」**
2. 제목 / 시작일 / 마감일 / 담당자 / 우선순위 / 활동 입력
3. 분류 = `휴가(연차)` 선택 시 시간 지정 가능 (반차/시간 휴가)
4. **「등록」**

### 5.3 프로젝트 단계 변경 (완료 처리)
1. 프로젝트 상세 헤더의 **「단계변경」** 클릭
2. **완료** 선택 → 완료일(default 오늘) 확인 → **「저장」**
   - 타절: 타절금액 + VAT 입력
   - 종결: 용역비/VAT가 자동 ₩0

## 7. 자주 묻는 질문

### Q. 「내 업무」가 비어있어요
본인 이름이 노션 담당자 옵션과 일치하지 않을 가능성이 큽니다. 우상단 프로필에서 본인 이름을 노션과 정확히 같은 형태로 수정.

### Q. 발주처를 입력했는데 노션에 안 들어갔어요
입력한 발주처 이름이 발주처 DB에 등록되어 있지 않으면 임시 텍스트 컬럼에 들어갑니다. 입력란 아래 **「발주처 DB에 추가」** 버튼을 눌러야 정식 relation으로 저장됩니다.

### Q. 휴가가 미분류로 보여요
이전 버그(2026-05-02 수정 완료). 모달에서 분류를 `휴가(연차)`로 다시 저장하면 정상 영역(일정 카드)으로 이동합니다.

### Q. 날인요청 등록 후 알림을 못 받았어요
받는 사람의 NAVER WORKS 계정이 시스템에 연결되지 않은 경우 발송이 skip됩니다. 관리자에게 본인 NAVER WORKS ID 등록 요청.

### Q. 자동 생성된 TASK가 보이지 않아요
이전 schema mismatch 버그(2026-05-02 수정). 새로 등록하시면 즉시 칸반에 표시됩니다.

### Q. 프로젝트가 노션에는 있는데 화면에서는 안 보여요
시스템은 노션과 5분 주기로 동기화합니다. 잠시 후 새로고침해보시고, 그래도 안 보이면 관리자에게 문의.

### Q. 화면이 멈추거나 502 에러가 나와요
강력 새로고침: `Ctrl + Shift + R`. 그래도 계속되면 1~2분 후 재시도. 안 풀리면 관리자에게 문의.

## 8. 권한 안내

| 역할 | 가능한 작업 |
|---|---|
| **일반 직원 (member)** | 본인 담당 프로젝트/업무·영업 관리, 발주처 read/추가/수정 (삭제 X), 날인요청 등록, 본인 항목 재날인요청, 주간업무일지 페이지·PDF 조회 (회사 전체 콘텐츠, 날인대장 포함) |
| **팀장 (team_lead)** | + 대시보드, 같은 팀 직원 업무 보기 (`/admin/employee-work`), 날인 1차 승인/반려, 프로젝트 일반 편집(이름·발주처·금액·phase 등) + 계약 분담 CUD |
| **관리팀 (manager)** | 대시보드 / 직원 일정 / 사용 매뉴얼 / 프로젝트 / 영업·발주처·수금·지출·계약서 관리. 프로젝트 일반 편집 + 계약 분담 + 수금 등록·수정·삭제 가능. 일반 직원 작업 영역(내 업무·날인·건의 등)과 시스템 관리는 미노출 |
| **관리자 (admin)** | + 모든 메뉴, 발주처 삭제, 프로젝트 단계 변경, 사용자 승인, 날인 2차 승인/반려, Drive 연결 관리, 주간업무일지 발행, 공지/교육 관리 |

자세한 권한 매트릭스는 [PERMISSIONS.md](PERMISSIONS.md) 참조.

## 9. 사이드바 메뉴 구조

역할에 따라 자동 노출/숨김. 관리자 그룹은 **펼침/접힘** 가능.

### 9.1 공통 (label 없음, 항상 노출)
- 대시보드 — admin / team_lead / manager
- 내 업무 — admin / team_lead / member
- 직원 업무 — admin / team_lead
- 직원 일정, 사용 매뉴얼 — 모두
